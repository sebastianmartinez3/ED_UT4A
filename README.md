# Actividad 4 de aula: Optimización y documentación

La Actividad 4 de aula: Optimización y documentación propone una práctica contextualizada en un entorno profesional ficticio en la que el alumnado debe mejorar un código legacy que funciona correctamente, pero presenta mala calidad interna.

El objetivo no es añadir funcionalidades, sino reducir la deuda técnica aplicando buenas prácticas de desarrollo:

- Uso de Git con ramas.
- Creación de pruebas unitarias con xUnit como red de seguridad.
- Refactorización sistemática del código.
- Documentación XML para mejorar su mantenibilidad y comprensión.

La actividad se desarrolla en cinco fases:

1. Preparación del entorno y control de versiones.
2. Creación de pruebas unitarias.
3. Refactorización aplicando patrones concretos (renombrado, encapsulación, constantes, guard clauses y extracción de métodos).
4. Autodocumentación.
5. Entrega final.

Como cierre, el alumnado debe subir el repositorio actualizado y entregar un informe en PDF breve y profesional donde explique el proceso seguido y proponga posibles mejoras futuras, partiendo de un código inicial con claros code smells incluido como anexo.
